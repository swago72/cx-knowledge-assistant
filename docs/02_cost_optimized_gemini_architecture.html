03 ‚Äî Components
6 Core System Components
COMPONENT 01
ingestion.py
Reads Google Play HTML files, strips nav/scripts with BeautifulSoup, chunks into 500-token segments, generates embeddings locally, and stores in ChromaDB with source metadata.
BeautifulSoup4 LangChain sentence-transformers ChromaDB
COMPONENT 02
rag.py
Embeds user questions with HuggingFace, performs cosine similarity search in ChromaDB, retrieves top-5 chunks, and passes them to Gemini with a citation-enforcing prompt.
LangChain RAG Gemini 2.0 Flash Prompt Engineering ConversationMemory
COMPONENT 03
analytics.py
Logs every query, response, source citation, and timestamp to SQLite. Renders a dashboard showing query volume, top cited docs, unanswered questions, and knowledge gaps.
SQLite Pandas Streamlit Charts
COMPONENT 04
app.py
Streamlit entry point. Two-panel chat UI with citations on the right. Document library sidebar, admin upload panel, and tab navigation between Chat and Analytics views.
Streamlit Session State Custom CSS
COMPONENT 05
.env
Single config file. Only one key needed ‚Äî your free Google AI Studio API key. No OpenAI account, no credit card, no paid services required anywhere in the stack.
GOOGLE_API_KEY python-dotenv
COMPONENT 06
requirements.txt

06 ‚Äî Build Timeline
6-Week Execution Plan
WEEK 01
Data Collection + ingestion.py
  • Save 14 Google Play support pages as HTML: refunds, billing, installs, payments, IAP
  • Set up environment: pip install langchain langchain-google-genai chromadb sentence-transformers streamlit beautifulsoup4 lxml pandas python-dotenv
  • Write ingestion.py ‚Äî BeautifulSoup parser, LangChain chunker, HuggingFace embedder, ChromaDB storage
  • Run ingestion ‚Äî verify ChromaDB stores chunks with source metadata
  • Milestone: Raw Python query returns relevant chunks from "how do I get a refund?"
WEEK 02
RAG Core ‚Äî rag.py
  • Build retriever ‚Äî HuggingFace embeds the question, ChromaDB returns top-5 chunks with source names
  • Build generator ‚Äî Gemini answers ONLY from retrieved chunks, citations enforced in system prompt
  • Add ConversationBufferWindowMemory ‚Äî last 5 turns of context maintained
  • Test with 20+ real Google Play support questions, tune chunk size and k-value
  • Milestone: "Why was I charged after cancelling?" returns a cited, accurate answer
WEEK 03
Analytics Layer ‚Äî analytics.py
  • Build SQLite logger ‚Äî log every query, answer, source doc, response time, timestamp
  • Build dashboard ‚Äî query volume by day, top 10 questions, most cited docs, unanswered queries
  • Add knowledge gap detection ‚Äî flag queries where no confident answer was found
  • Milestone: Dashboard shows a clear story about which docs get used most and where gaps exist
WEEK 04
Streamlit UI ‚Äî app.py
  • Build two-panel chat layout: conversation on left, source citations on right
  • Add document library sidebar showing all 14 ingested HTML docs
  • Add admin upload tab ‚Äî drag-and-drop new HTML files, re-index button
  • Wire analytics.py into a dedicated Analytics tab
  • Milestone: A non-technical person can use the full app end-to-end without help
WEEK 05
Polish + Testing

  • Edge case testing ‚Äî irrelevant questions, empty results, very long queries
  • UI polish ‚Äî loading spinners, copy-to-clipboard, timestamps, mobile responsiveness
  • Milestone: App handles gracefully anything a real support agent might throw at it
