03 ‚Äî Components
6 Core System Components
COMPONENT 01
Document Ingestion Pipeline
Reads PDFs and Word docs, chunks them into 500-token segments with overlap, generates vector embeddings, and stores them in ChromaDB with source metadata.
LangChain PyPDF2 python-docx OpenAI Embeddings ChromaDB
COMPONENT 02
RAG Query Engine
Embeds user questions, performs semantic search against vector store, retrieves top-k relevant chunks, and passes them to the LLM with a strict citation-enforcing prompt.
LangChain RAG GPT-3.5-turbo Prompt Engineering FastAPI
COMPONENT 03
Conversational Memory
Maintains context across multi-turn conversations so agents can ask follow-up questions without re-explaining. Uses a sliding window buffer to stay within token limits.
ConversationBufferMemory LangChain Token Management
COMPONENT 04
Chat UI + Citation Panel
Clean agent-facing chat interface with a source citation panel showing exactly which SOP section each answer came from, plus a document library sidebar and confidence indicator.
Streamlit Custom CSS Session State
COMPONENT 05
Analytics Dashboard
Tracks query volume, most-cited documents, unanswered questions, and response quality scores. Surfaces knowledge gaps where agents can't find answers ‚Äî your WFM analytics DNA lives here.
Streamlit Charts SQLite Pandas Power BI (optional)
COMPONENT 06
Admin Panel
Upload new SOP documents, re-index the knowledge base, view document inventory, and manage chunking settings ‚Äî without touching code. Makes the system self-serviceable.
Streamlit File Uploader ChromaDB API Batch Embeddings

06 ‚Äî Build Timeline
6-Week Execution Plan
WEEK 01
Data Collection + Ingestion Pipeline
  • Download 30‚Äì50 PDFs from cms.gov and cfpb.gov (Medicare handbooks, complaint handling guides)

  • Write loader.py, chunker.py, embedder.py
  • Run ingestion ‚Äî verify ChromaDB stores chunks with metadata
  • Milestone: raw Python query returns relevant chunks from a test question
WEEK 02
RAG Core ‚Äî Retrieval + Generation
  • Build retriever.py ‚Äî semantic search returning top-5 chunks with source doc name + page
  • Build generator.py ‚Äî GPT-3.5 prompt that answers ONLY from provided chunks
  • Write prompts.py ‚Äî citation-enforcing system prompt
  • Test with 20+ real questions, tune chunk size and k-value
  • Milestone: Ask "what is the escalation process for a prior auth dispute?" and get a cited answer
WEEK 03
Conversational Memory + API Layer
  • Add ConversationBufferWindowMemory ‚Äî maintain last 5 turns of context
  • Wrap everything in FastAPI endpoints: POST /query, GET /documents, POST /upload
  • Build db/logger.py ‚Äî log every query, response, source, timestamp to SQLite
  • Milestone: Multi-turn conversation works; follow-up questions use prior context correctly
WEEK 04
Streamlit UI ‚Äî Chat Interface
  • Build ui/chat.py ‚Äî clean two-panel layout: chat on left, citations on right
  • Add document library sidebar showing all ingested SOPs
  • Add typing indicator, message timestamps, copy-to-clipboard on answers
  • Build ui/admin.py ‚Äî drag-and-drop document upload, re-index button
  • Milestone: Someone with no technical knowledge can use the UI end-to-end
WEEK 05

  • Unanswered queries list ‚Äî questions where confidence was low (your WFM instinct)
  • Response time distribution ‚Äî system performance monitoring
  • Milestone: Dashboard tells a story about knowledge base health and agent behavior
